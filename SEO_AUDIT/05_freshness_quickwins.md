## 5-A. 總體 Freshness 危機

| 指標 | 數值 |
|------|------|
| 全站有 `updated` frontmatter 欄位 | **0 篇**（全站） |
| `dateModified` 等同 `datePublished` | **693 篇** |
| 12 個月內新發布（2025-05-18 後） | 270 篇 |
| 超過 12 個月、常青主題、無更新記錄 | **389 篇** |
| 標題含「2024」但未升至 2026 | **19 篇** |
| 標題含「2025」但未升至 2026 | **25 篇** |

**核心問題**：Google 判斷文章是否「新鮮」主要看 `dateModified`（Schema.org），而全站 dateModified 等同 pubDate，即便文章內容有更新，Google 也看不到。這讓 mimigo.tw 的「2026 最新版」標題操作直接吃掉你在搜尋結果的 CTR。

## 5-D. 如何正確設定 `updated` 欄位

### 步驟 1：在 frontmatter 加入 `updated` 欄位

```markdown
---
title: "佩尼達島全攻略"
pubDate: "2023-08-16"
updated: "2026-05-18"   ← 新增這行
---
```

### 步驟 2：確認 BlogPost.astro 傳入 updatedDate

已確認 `src/layouts/BlogPost.astro` 有以下邏輯：
```astro
const { ..., updatedDate, ... } = Astro.props;
// 已傳入 SEO 元件
modifiedTime={updatedDate ? new Date(updatedDate).toISOString() : ...}
```

`src/content/config.ts` 中的 schema 是否允許 `updated` 欄位需確認（如不存在需新增）。

### 步驟 3：確認 content config 有此欄位

```typescript
// src/content/config.ts 應包含：
updated: z.string().optional(),
// 或
updatedDate: z.coerce.date().optional(),
```

## 5-E. 30 篇更新的最小可行動作（每篇 15–30 分鐘）

| 類型 | 動作 | 效果 |
|------|------|------|
| 年份標題更新 | 標題 2024/2025 → 2026 | CTR 提升（mimigo 操作的核心） |
| 加一段「2026 最新更新」 | 在文章底部加 200 字最新資訊 | 實質 fresh content |
| 加地圖連結 | 插入換匯所/ATM/地圖 CTA | 地圖 entity 強化（階段 3） |
| `updated` 欄位 | frontmatter 加一行 | dateModified 訊號立刻生效 |

**每篇平均工時**：15 分鐘  
**30 篇總計**：約 7–8 小時  
**預期效果**：fresness 訊號啟動，標題年份競爭力恢復，高流量文章重新獲得排名機會
